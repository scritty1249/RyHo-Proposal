Reasoning for Changes

First Steps

Directory map as of 04/23/2024 at 02:23 AM (PST)

To streamline the process should this ever have to be done again, and because of the absence of any existing documentation, I set about creating a tool to get a visual map of the existing Drive structure. Using this tool to remove redundancies, forms and their linked spreadsheets have been collected into standalone folders, isolated from the rest of the Drive. The goal is to make it easier to restrict access to more sensitive information as needed, without compromising ease of use for key staff.

Structural Changes

See FIle Hierarchy Documentation for notes and documentation on the Drive structure.

Google Forms have all been separated into a designated folder following the path,

/Operations/Forms/

and have been separated further between forms used by those outside of the company, and those within.

/Operations/Forms/External/
/Operations/Forms/Company/

The corresponding Form Responses are stored in alongside the Forms/ folder in the path:

/Operations/Form Responses/

The structure each response is stored in mirrors the Forms they are linked to. This should make it easier to find the corresponding data for any Forms in the future.
Form Response Spreadsheets only contain one Sheet- for the corresponding Form. The name of the Sheet must be "Form Responses".

/Operations/Form Responses/External/
/Operations/Form Responses/Company/

Dashboards/Interfaces have been separated into their own Spreadsheets. The data containing calculations for each are stored in,

/Operations/Information Technology/Dashboard & Interface Calculations

Organizational Intent:

  • ALL information can be found in this central folder, no matter what. References(Shortcuts) to these files can be placed anywhere, but the sources should remain here.
  • Keeping everything in one place should not only make things easier to find, but also simplify migrating data in the future
  • Prevents accidental deletions. NOTHING should be deleted in these folders unless management gives the OK.
    Security Intent:
  • When needed, view access to specific information can be easily revoked while retaining access to information critical to job functions. (Example- Currently, people are unable to interact with the Financial Data interface without full write access to all Timesheet logs. This could be very bad if someone were to tamper with the logs.)

Form changes

To maintain consistency and clarity the following names have been edited, alongside any connected spreadsheets and response folders:

  • Interview Availability -> Interview Availability Form
  • Applicant Scorecard -> Applicant Scorecard Form
  • Sales Report 2024 -> Sales Report 2024 Form
  • Disbursement Form -> Advance Payment Disbursement Form
  • Request Form -> Advance Payment Request Form
  • Reconciliation Form -> Advance Payment Reconciliation Form

I edited or added response validation to the following:

  • Email formatting validation for Interview Availability Form, Onboarding Form and RyHo Rentals Application Form to reject any input that does not match the pattern of a valid email address. All email formatting validation follows the same Regular Expression pattern:

    ^([^\s]+)@([^\s]+)\.([^\s\.]+)$ <- (Google uses the RE2, or "Golang" RegEx Engine)
    *This pattern specifically searches for any length of any non-space characters surrounding and between an ampersand and a period.

  • Lunch break length fields in the On-Site Checkout Form and Timesheet Form have been set to specifically take a duration value following the HH:MM:SS format. Field descriptions have been included in the affected forms to clarify the format.

  • Input fields for dollar amounts in the On-Site Checkout Form, Sales Report 2024 Form, Equipment Liquidation Form, Disbursement Form, Request Form, and Reconciliation Form have been set to only accept number values at or greater than zero.

Intent for Form Validation changes:

  • I have repeatedly seen formulas break from erroneous user input or lack of clarity in form descriptions (such as what units of time to use, hours vs minutes). I believe the problem can be rendered null be solved by indicating in the forms that only specific types of information are acceptable (Example: do not put words in a field that is supposed to be a number)

More significant modifications:

  • The Timesheets Form question for "Lunch Duration" has been replaced with "Lunch Break?", and "Lunch In" and "Lunch Out" have been moved to a new section called "Lunch Break". The question "Did you pay for something that needs reimbursement?" has been moved to a new section called "Reimbursement?" but otherwise functions the same.

Spreadsheet Changes

To clarify for those editing sheets later, bottom borders in the header row following a color code based on content and intended purpose:

  • Orange - columns for users to change values (Example: checkboxes)
  • Blue - columns that reference data from other Spreadsheets
  • Yellow - columns that contain formulas which do not reference other spreadsheets
  • Grey - other

To clarify data being handled by formulas, all columns that use distinctive formatting (such as dollar formatting and time formatting) have their header's top border colored green.

Google Apps Scripting

To facilitate the function of dashboards and interfaces with minimal manual intervention, I have used Google Apps Scripts to automate the creation of certain rows. See Google Apps Scripting Documentation for notes and documentation.
